`evlog/better-auth` automatically identifies users on every request. Every wide event includes who made the request — no manual `log.set({ user })` needed.

::code-collapse

```txt [Prompt]
Add Better Auth user identification to my app with evlog.

- Import createAuthMiddleware from 'evlog/better-auth'
- Use createAuthMiddleware(auth) in a server middleware to auto-identify users on every request
- Use useAuthIdentity() on the client to sync identity to client-side logs
- Works with all frameworks: Nuxt, Express, Hono, Fastify, NestJS, Elysia, standalone

Docs: https://www.evlog.dev/logging/better-auth
Adapters: https://www.evlog.dev/adapters
```

::

## Quick Start (Nuxt)

Create a server middleware that resolves the Better Auth session on every request:

```typescript [server/middleware/auth-identify.ts]
import { createAuthMiddleware } from 'evlog/better-auth'
import { auth } from '~/lib/auth'

const identify = createAuthMiddleware(auth)

export default defineEventHandler(async (event) => {
  if (event.path.startsWith('/api/auth/')) return
  if (!event.context.log) return

  await identify(event.context.log, event.headers)
})
```

Every wide event now includes the user:

```json [Wide Event]
{
  "method": "POST",
  "path": "/api/checkout",
  "status": 200,
  "duration": "120ms",
  "userId": "usr_123",
  "user": {

    "name": "Hugo Richard",
    "email": "hugo@example.com",

    "createdAt": "2024-01-15T10:00:00.000Z"
  },
  "session": {
    "id": "sess_abc",
    "expiresAt": "2024-01-22T10:00:00.000Z"
  }
}
```

## How It Works

The integration resolves the Better Auth session early in the request lifecycle (before evlog strips sensitive headers), calls `identifyUser()` to set safe user and session fields on the logger, and skips auth routes by default.

| Function | Description |
|----------|-------------|
| `identifyUser(log, session)` | Core helper — extracts safe fields from a session and sets them on the logger |
| `createAuthIdentifier(auth)` | Nitro `request` hook factory — auto-identifies users on every request |
| `createAuthMiddleware(auth)` | Framework-agnostic — returns an async `(log, headers) => Promise<void>` function |

## identifyUser

The core building block. Takes a `RequestLogger` and a Better Auth session, extracts safe fields, and calls `log.set()`:

```typescript
import { identifyUser } from 'evlog/better-auth'

const session = await auth.api.getSession({ headers: event.headers })
if (session) {
  identifyUser(log, session)
}
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maskEmail` | `boolean` | `false` | Mask emails as `h***@example.com` |
| `session` | `boolean` | `true` | Include session metadata |
| `fields` | `string[]` | `['id', 'name', 'email', 'image', 'emailVerified', 'createdAt']` | Whitelist of user fields to extract |

```typescript
identifyUser(log, session, {
  maskEmail: true,
  fields: ['id', 'name'],
  session: false,
})
```

## createAuthIdentifier (Standalone Nitro)

A factory that creates a Nitro `request` hook. Designed for standalone Nitro apps (without Nuxt) where the evlog Nitro module handles both the logger and hook ordering.

::note
For **Nuxt**, use `createAuthMiddleware` in a server middleware instead — Nitro plugin hook ordering can cause the logger to not be available yet in the `request` hook.
::

```typescript [server/plugins/evlog-auth.ts]
import { createAuthIdentifier } from 'evlog/better-auth'
import { auth } from '~/lib/auth'

export default defineNitroPlugin((nitroApp) => {
  nitroApp.hooks.hook('request', createAuthIdentifier(auth, {
    maskEmail: true,
    exclude: ['/api/auth/**', '/api/public/**'],
  }))
})
```

### Options

Inherits all `identifyUser` options, plus:

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `exclude` | `string[]` | `['/api/auth/**']` | Route patterns to skip (glob) |
| `include` | `string[]` | `undefined` | If set, only matching routes are resolved |

The default excludes `/api/auth/**` to avoid resolving sessions during auth flows (the session may not exist yet).

### Performance

`getSession()` costs a database query unless Better Auth's [cookie caching](https://www.better-auth.com/docs/concepts/session-caching) is enabled. For high-traffic apps:

1. Enable Better Auth session caching
2. Use `exclude` to skip public routes that don't need user context
3. Use `include` to limit resolution to specific routes

## createAuthMiddleware (any framework)

For non-Nitro frameworks, use `createAuthMiddleware` to get a reusable identify function:

::code-group
```typescript [Express]
import { createAuthMiddleware } from 'evlog/better-auth'
import { auth } from './lib/auth'

const identify = createAuthMiddleware(auth, { maskEmail: true })

app.use(async (req, res, next) => {
  await identify(req.log, req.headers)
  next()
})
```

```typescript [Hono]
import { createAuthMiddleware } from 'evlog/better-auth'
import { auth } from './lib/auth'

const identify = createAuthMiddleware(auth)

app.use(async (c, next) => {
  await identify(c.get('log'), c.req.raw.headers)
  await next()
})
```

```typescript [Fastify]
import { createAuthMiddleware } from 'evlog/better-auth'
import { auth } from './lib/auth'

const identify = createAuthMiddleware(auth)

app.addHook('onRequest', async (request) => {
  await identify(request.log, request.headers)
})
```

## Client Identity Sync

On the client side, watch the Better Auth session and call `setIdentity()` to include user context in client logs:

```typescript [composables/useAuthIdentity.ts]
import { authClient } from '~/lib/auth-client'

export function useAuthIdentity() {
  const session = authClient.useSession()

  watch(() => session.value?.data?.user, (user) => {
    if (user) {
      setIdentity({ userId: user.id, userName: user.name })
    } else {
      clearIdentity()
    }
  }, { immediate: true })
}
```

Call it once in your root layout:

```vue [app.vue]
<script setup>
useAuthIdentity()
</script>
```

Client-side logs now include the user identity:

```json [Client Log]
{
  "level": "info",
  "tag": "checkout",
  "message": "User clicked checkout",
  "userId": "usr_123",
  "userName": "Hugo Richard"
}
```

## Wide Event Fields

| Field | Source | Description |
|-------|--------|-------------|
| `userId` | `session.user.id` | Top-level user ID (used by PostHog adapter as `distinct_id`) |
| `user.id` | `session.user.id` | User ID |
| `user.name` | `session.user.name` | Display name |
| `user.email` | `session.user.email` | Email (maskable) |
| `user.image` | `session.user.image` | Avatar URL |
| `user.emailVerified` | `session.user.emailVerified` | Email verification status |
| `user.createdAt` | `session.user.createdAt` | Account creation date |
| `session.id` | `session.session.id` | Session ID |
| `session.expiresAt` | `session.session.expiresAt` | Session expiry |
| `session.ipAddress` | `session.session.ipAddress` | Client IP |
